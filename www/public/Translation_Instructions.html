Java SE API Specification Translation Guidelines

The following guideline explains the format of the comment (.cmnt) files and the rules to follow when translating these files.


Comment File Format

Each comment file consists of zero or more blocks of the following form:

<!-- BEGIN KEY -->
Key
<!-- END KEY -->
<!-- BEGIN DATA -->
Data
<!-- END DATA -->

Key:  consists of the signature of the identifier (class, interface or member), terminated with a newline, and should not be translated.
Data: consist of any number of strings terminated with newlines, and comprises the documentation comment for translation.



CMNT File Example

The following is an example of a comment (.cmnt) file. 

Copyright 2006 Sun Microsystems, Inc. All rights reserved.
Use is subject to 'license terms' given in
"https://jdk-api-zh-cn.dev.java.net/public/Java_SE_5_API_Specification_License.pdf"

<!-- BEGIN KEY -->
public boolean java.util.Collection.retainAll(java.util.Collection<?> c)
<!-- END KEY -->
<!-- BEGIN DATA -->
Retains only the elements in this collection that are contained in the specified
collection (optional operation).  In other words, removes from this collection all
of its elements that are not contained in the specified collection.
@param c elements to be retained in this collection.
@return <tt>true</tt> if this collection changed as a result of the call
@throws NullPointerException if the specified collection is <tt>null</tt>.
@see #remove(Object)
@since 1.2
<!--END DATA-->



Translation Rules

RULE 1
The first paragraph in each comment (.cmnt) file contains the copyright and the license information.
This paragraph should NOT be translated.

RULE 2
The string tags <!-- BEGIN KEY --> and <!-- END KEY -->  should NOT be translated.

RULE 3
The strings between <!-- BEGIN KEY --> and <!-- END KEY -->  should NOT be translated.
Even one extra space will cause a failure in javadoc generation.

RULE 4
The string tags <!-- BEGIN DATA --> and <!-- END DATA --> should NOT be translated.

RULE 5
The strings between <!-- BEGIN DATA --> and <!-- END DATA --> should be translated.

RULE 6
The strings(tags) with "@" should NOT be translated even if they are between <!-- BEGIN DATA --> and <!-- END DATA -->

RULE 7
The phrase after the string(tags) with "@" are translated or not translated depending on the tag.

The following tag value should be translated
@return value      ---> value should be translated.
@event value       ---> value should be translated.
@param value     ---> value should be translated.
@exception value ---> value should be translated.
@throws value     ---> value should be translated.
@deprecated        ---> value should be translated.

The following tags value should NOT be translated:
@author value     ---> value should not be translated.
@see value          ---> value should not be translated.
@since value       ---> value should not be translated.
@version value    ---> value should not be translated.
{@link value}     ---> value should not be translated.

Example 7.1
@return The height of the image data in pixels.
  The returned value should be translated.
 Example 7.2
@event the key event to be handled
  The event value should be translated.
Example 7.3
@param bi A BufferedImage that is similar to the BufferedImage(s)
@param options the integer bit flag options.
The parameter value should be translated though, the word just after @param (for example "bi" or "options") should NOT be translated.
"bi" or "options" should be consistent with the parameter specified in the signature (Key).
Example 7.4
@exception IllegalArgumentException thrown if the tableNum ...
  The exception value should be translated though, the word just after @exception (IllegalArgumentException) should NOT be translated.
Example 7.5
@throws ObjectCollectedException if this object has been ...
  The exception to be throwed: should be translated, though, the word just after @throws (ObjectCollectedException) should  NOT be translated.
Example 7.6
@deprecated This class implements a deprecated interface, ...
  Thedeprecated value should be translated
Example 7.7
@author Tom Sausner
The author should NOT be translated
Example 7.8
@see JPEGImageEncoder
  The see value  should NOT be translated

Example 7.9
@since JDK1.2
The JDK version of implementation: should NOT be translated
Example 7.10
@version 4 December 1997
The version value should NOT be translated
Example 7.11
For {@link ArrayType}, the return value depends on the array component type.
       The inline link value should be translated
       The words within {} should NOT be translated.
        In case that the strings are specified as follows, "<code>backing array</code>", the string (for example "backing array") should be translated.
        It will have a {@link #array <code>backing array</code>} ...

RULE 8
The first line of the description of each package, class, interface, method, etc. is used for other table or index. The line is recognized by a period (.)  The first line should be translated as it is.
